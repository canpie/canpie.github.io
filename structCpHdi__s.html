CpHdi_s Struct Reference

Detailed Description

The Hardware Description Interface provides a method to gather information about the CAN hardware and the functionality of the driver. The hardware description structure is available for every physical CAN channel.

uint8_t CpHdi_s::ubBufferMax

◆ ubCanFeatures

uint8_t CpHdi_s::ubCanFeatures

The element ubCanFeatures defines the capabilities of the CAN controller.

  • Bit 0: 0 = 2.0A support, 1 = 2.0B support
  • Bit 1: 0 = Classic CAN , 1 = ISO CAN FD
  • Bit 2: reserved
  • Bit 3: reserved
  • Bit 4: reserved
  • Bit 5: reserved
  • Bit 6: reserved
  • Bit 7: reserved

◆ ubDriverFeatures

uint8_t CpHdi_s::ubDriverFeatures

The element ubDriverFeatures defines the capabilities of the software driver.

  • Bit 0: 0 = no time stamp, 1 = time stamp support
  • Bit 1: 0 = no user data , 1 = user data support
  • Bit 2: reserved
  • Bit 3: reserved
  • Bit 4: reserved
  • Bit 5: reserved
  • Bit 6: reserved
  • Bit 7: reserved

◆ ulBitrate

uint32_t CpHdi_s::ulBitrate

The element ulBitrate defines the actual configured bit-rate of the CAN controller in bits-per-second (bps). For ISO CAN FD the value defines the bit-rate of the arbitration phase.

◆ ulBitrateFD

uint32_t CpHdi_s::ulBitrateFD

The element ulBitrateFD is only valid for ISO CAN FD controller. The value defines the actual configured bit-rate of the data phase in bits-per-second (bps).
